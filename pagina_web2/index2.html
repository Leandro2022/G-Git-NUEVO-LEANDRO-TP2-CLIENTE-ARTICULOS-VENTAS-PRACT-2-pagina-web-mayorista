SISTEMA VENTA MAYORISTA version 1.0

EL SISTEMA DE MERCADO MAYORISTA, le será útil si se desea llevar un control del cliente y el artículo que vende el mercado. El sistema tendra un ingreso de usuario y contraseña para el personal de caja. El administrador sera el encargado de registrar los artículos, como también a los usuarios al sistema. La cajera tendrá la actividad de pedir el DNI del cliente para registrar en la sección ventas con sus artículos que el cliente eligió para su compra. La salida será dada un ticket con el total de compras, los datos del cliente y la fecha.



OBJETIVO GENERAL DEL SISTEMA VENTA MAYORISTA

El objetívo del sistema le facilítara la registración de artículos para el depósitor/ar y a la cajera/ro para la venta de los artículos Las actividades que se destacan son:

  • Registrar clientes con sus datos al sistema.
  • Registrar los artículos al sistema.
  • Generar un informe general de las compras que hicieron los clientes en el día.
  • Generar un documento que será entregado al cliente con sus compras realizadas.

    REGISTRAR USUARIO NUEVO AL SISTEMA

    El sistema de VENTA MAYORISTA tendra un ingreso de seguridad para los usuarios, pero antes de ingresar debera registrarse a la base de datos. El administrador de la empresa tendra una clave de acceso para ingresar al campo de "REGISTRAR USUARIO NUEVO". El administrador le tomara una entrevista al nuevo empleado cajera/o que, una vez hecha la entrevista el administrador le pedira un USUARIO Y CONTRASEÑA al nuevo empleado. El empleado le dira un usuario y contraseña y el administrador le agrega al sistema. Esos datos del empleado queda registrado a la base de datos del sistema. Y luego de todo ese proceso el empleado puede empezar a trabajar en su puesto de trabajo.

    REGISTRAR CLIENTE AL SISTEMA

    En esta aparte tenemos la registracion de un cliente, que el administrador del sistema le dara un alta al usuario antes de hacer una compra.

    Los datos que pondran son los siguientes:

  • NOMBRE DEL CLIENTE
  • APELLIDO DEL CLIENTE
  • DNI DEL CLIENTE (solo acepta numeros)
  • E-MAIL DEL CLIENTE
  • TELEFONO DEL CLIENTE
  • DIRECCION DEL CLIENTE
  • LOCALIDAD DEL CLIENTE
    •

    Luego de agregar todos estos datos mencionados se les dara el alta cliente y automaticamente se le guarda en el sistema de la base de datos.

    Para modificar un dato del cliente sean: E-MAIL, TELEFONO, DIRECCION, LOCALIDAD que seran los unicos datos que se podra modificar. una ves modificado estos datos mencionados le da seleccion en el boton modificar y automaticamente se modifica los datos y sale un mensaje en pantalla que fueron los datos modificados.

    REGISTRAR ARTÍCULOS

    En registración de artículos tenémos algo similar que en registración de cliente. En esta parte tambien manejara el administrados o empaquetador de la empresa.Acá tenemos menos datos, los datos son: NOMBRE DEL ARTICULO, MARCA DEL ARTICULO, PRECIO DEL ARTICULO, CANTIDAD DE ARTICULOS. Para dar de alta un artículo debemos hacer el mismo procedimiento de alta cliente. Debemos agregar todos los datos del artículo y seleccionar el botón alta y se guarda automaticamente a la base de datos del sistema. Para dar modifición de un articulo. lo que se modificara sera lo siguiente. PRECIO y CANTIDAD. seran los unicos datos que se podra modificar. Luego de modificar estos datos mencionados podra seleccionar el boton de modificar y sale un mensaje que los datos ya estan modificados.

    SECCIÓN VENTAS

    En esta sección tenemos lo mas importante del sistema ya que sera la salida de los procesos, aca tenemos la parte de DATOS DEL CLIENTE el cajero/a le pedirá el DNI del cliente antes de hacer una venta. Una vez que le pidio el DNI al cliente le da seleccion al boton BUSCAR POR DNI y automaticamente en donde esta NOMBRE APELLIDO Y DIRECCION se completa automaticamente los datos. En caso que no esten registrado le saldra un mensaje por pantalla que le pedira que le agrege al sistema y le da un ENTER y le lleva a la ventana de REGISTRAR CLIENTE AL SISTEMA y le pedira los datos completos. en caso que no tenga TELEFONO O E-MAIL que ponga el siguiente signo ( - ). Una vez buscado el cliente por DNI se le habilita la parte de ARTICULO, en donde se hace la venta. Para el articulo debemos ingresar el ID DEL ARTICULO sera un número que esta en cada paquete. ejemplo: 1.2.3.4.5...... Una vez que agrego el ID debe DAR UN ENTER para que cargue los datos del articulo como NOMBRE, MARCA y PRECIO. En caso que no este el articulo en stok le saldra un mensaje en pantalla que lo informe al proveedor o administracion. Al darle aceptar le llevara a la registracion de articulos. Si esta en stok del articulo abajo de ID ARTICULO esta la CANTIDAD. Aquí debe poner la cantidad que de articulo que se lleva el cliente, ejemplo: 1.2.3 de "HARINA" Y luego de todos estos pasos le da en COMPRAR y sale un mensaje de que la compra fue realizado y con el nombre del articulo. Esta el ticket con la venta que realizo, sera dado al cliente impreso con sus datos y datos del articulo y con su total. El informe es general. es un documento con todas las ventas realizadas en el día, semanas, mes y año. Para eliminar o limpiar todos los campos debe seleccionar en SIGUIENTE CLIENTE y se limpia listo para otra venta.
